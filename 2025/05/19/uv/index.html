1. uv

Rust-based package and project manager that replace pip, pip-tools, venv, and potentially poetry.

1. install and initialize a new project

  • install:

  • Creates pyproject.toml (Initializes a new project)

  • Create venv other than a project

    • (this is not needed when create a project with uv init), only for creating a venv to run some scripts
    • no pyproject.toml file will be created

  • Python version

2. Setup cloned project:

  • like the pip install, should be executed after git clone a new project.

  • steps:

    1. looks at pyproject.toml to get all requested dependencies
    2. checks if uv.lock matches those requests
    3. if match: uses uv.lock as the source of truth to install the exact dependencies.
    4. if not match (like pyproject.toml was edited manually): re-resolve dependencies based on pyproject.toml, update the uv.lock, then install

  • options

    • –frozen: use uv.lock without checking pyproject.toml, if uv.lock is missing, the command fails.
    • –locked: use uv.lock with double-check if it’s still in sync with pyproject.toml, if out of sync, an wrror will be trhow instead of updating the lockfile

3. Denpendencies management:

  • uv add

    • whenever working on a prject has a pyproject.toml, this command should be used to add dependencies
    • pyproject.toml, uv.lock, .venv all will be took care.

2. Python package

Module:

  • A single python file (.py) containing code (functions, classes, variables)

Package:

  • A collection of modules(.py files) organized in directories,
  • Recognized by the presence of an init.py file (usally it is empty) within each directory.
  • Packages can contain sub-packages.

Dependency:

  • An extenal python package (or a specific version of it).

  • Dependency Specification:

    • requirements.txt:

      • A plain text file listing direct dependencies,
      • Typically generated using pip freeze.
      • It often lists all transitive dependencies, making it hard to manage top-level requirements.

    • pyproject.toml:

      • Modern, standardized way to declare project metadata and dependencies.
      • A TOML(Tom’s Obvious, Minimal Language) file that serves as a central configuration.
      • Usually only direct dependencies.

    • Lock file(uv.lock,poetry.lock, etc.):

      • A file generated by a package manager (like uv or poetry) that lists all direct and transitive dependencies with their exact, pinned versions and hashes.
      • Ensures absolute reproducibility of the environment.

Virtual Environment:

  • An isolated python environment that allows to install packages for a specific project without interfering with other projects or the global pthon installation.
  • Most crucial concept for robust python development.
